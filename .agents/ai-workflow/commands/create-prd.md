# Create PRD Workflow

Use this before greenfield builds or substantial feature work.

## Objective

Create or update `PRD.md` so implementation starts from aligned requirements, not assumptions.

## Discovery

Ask targeted questions when any of these are unclear:

- User and problem.
- MVP scope.
- Non-goals.
- Tech stack constraints.
- Data model and integrations.
- Authentication, authorization, privacy, and security needs.
- Deployment target.
- Success criteria.

Use web or codebase research when current external facts, libraries, APIs, or examples matter.

## Required PRD Sections

1. Executive Summary
2. Mission
3. Target Users
4. MVP Scope
5. Explicit Non-Goals
6. User Stories
7. Architecture And Patterns
8. Technology Stack
9. Security And Configuration
10. Data Model Or API Specification
11. Success Criteria
12. Implementation Phases
13. Validation Strategy
14. Risks And Mitigations
15. Future Considerations

## Implementation Phases

Each phase must include:

- Goal.
- Deliverables.
- Files or modules likely affected.
- Validation criteria.
- Exit condition.

## Output

Write the PRD to `PRD.md` unless the user specifies another path. Summarize assumptions and the first recommended phase.
