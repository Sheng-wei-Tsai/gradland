In a route handler, replace your current upload logic: `await files.upload('resumes/${userId}/${filename}', fileBlob, { contentType: 'application/pdf' });` — returns a typed result with the public URL.,4. For Claude AI routes, import the pre-built tools: `import { createFileTools } from 'files-sdk/claude'; const tools = createFileTools(files);` then pass `tools` directly into your `client.messages.create({ tools, ... })` call — no manual tool schema writing.,5. Access the native client any time via `files.raw` if you need provider-specific features like Vercel Blob's `put` options or R2 bucket metadata.

Use `files.signedUploadUrl()` to generate a short-lived client-side upload URL — the browser uploads directly, bypassing your API server entirely, which cuts cold-start timeouts on large PDF uploads.
2. AI-powered resume diff tool: store every version of a user's resume under a consistent key pattern (`resumes/{userId}/{timestamp}.pdf`), use `files.list('resumes/{userId}/')` to enumerate versions, then feed two versions into Claude Sonnet with a 'what changed and did it improve?' prompt — gives users a changelog of their own resume evolution over their job search.
3. Claude file tools for the interview prep route: wire `createFileTools(files)` into your `/api/interview` Claude calls so the model can autonomously fetch the user's stored resume PDF mid-conversation, extract relevant experience bullets, and ground its follow-up questions in the user's actual background — no manual context-stuffing needed in the prompt.
