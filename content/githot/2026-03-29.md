## 3. [HKUDS/OpenSpace](https://github.com/HKUDS/OpenSpace)

**1,961 stars this week** · Python

OpenSpace is a shared memory/skill layer that lets AI coding agents (Claude Code, Codex, Cursor, etc.) learn from past task solutions and share that knowledge across sessions and agents, cutting token usage by ~46%.

**Use case**

Every time you run Claude Code or Codex on a new task, the agent reasons from scratch — even if you've solved a nearly identical problem before. OpenSpace intercepts that process, retrieves relevant past solution patterns from a shared vector store, and injects them as context. Concrete example: you've already debugged a Supabase RLS policy conflict — next time any agent hits a similar auth error, OpenSpace surfaces that solution instead of burning 10K tokens re-exploring it.

**Why it's trending**

It dropped the same week OpenAI Codex CLI and Claude Code went mainstream as terminal-native agents, hitting a real pain point developers are actively feeling right now: these tools are powerful but stateless and expensive at scale. The '$11K earned in 6 hours' hook from a Cursor competition also drove significant social media pickup.

**How to use it**

1. Install: `pip install openspace-agent` (Python 3.12+ required) and run `openspace init` to scaffold a local skill store.
2. Connect your agent: wrap your existing Claude Code or Codex invocations with `openspace --query "your task description"` — it retrieves matching past solutions before the agent starts.
3. After a successful task, OpenSpace auto-logs the solution pattern to the shared store: `openspace commit --task "fixed Supabase RLS infinite recursion" --solution ./solution_notes.md`.
4. Query the store directly for debugging: `openspace search "next.js hydration mismatch"` returns ranked past solutions with token cost metadata.
5. Optionally connect to the community cloud at open-space.cloud to pull anonymized skill patterns from other developers hitting the same stack.

**How I could use this**

Over time your cover letter agent stops starting from scratch and instead retrieves 'this fintech startup responded well to this structure' — measurably improving hit rate without manual prompt engineering.
3. Add OpenSpace as the skill backend for your blog's AI chat assistant — instead of pure RAG over your posts, the assistant retrieves from a dual store: your blog content AND a skill store of 'how Henry debugged/built X', letting readers ask implementation questions and get answers grounded in your actual past solutions rather than hallucinated generics.
