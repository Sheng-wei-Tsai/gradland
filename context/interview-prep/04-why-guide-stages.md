│   ┌────────────────────────────────────────────────────────────┐       │
│   │  1. The timing difference (the key distinction)            │       │
│   │  2. A concrete example — when you'd choose each one        │       │
│   │  3. Why it matters in practice (mention DOM / paint)       │       │
│   └────────────────────────────────────────────────────────────┘       │
│                                                                        │
│   If you can add a code snippet mentally — even just 2 lines —         │
│   that separates you from every other candidate who just reads         │
│   documentation back.                                                  │
│                                                                        │

│                                                                        │

│   │  useEffect        →  runs AFTER browser paints             │       │

## Why Each Element

| Element | Decision | Why |
|---------|----------|-----|
| Numbered framework box | `3-beat structure` in a bordered inset | Gives the user a memorisable scaffold — not a wall of text |
| Code/concept diff box | Two-line contrast table | Visual anchoring — the most important concept in the most scannable format |
| "I've seen candidates fail..." | First-person voice | Alex is a mentor who has been in the room — this builds authority and makes advice feel real |
| ✅ green callout | Highlights the signal interviewers look for | Shows the user the interviewer's mental model, not just the answer |
| XP on the button | `[ Understood → +10 XP ]` | Reward is tied to the action — immediate reinforcement |
| Stage label 🧠 / 🗺️ | Replaces numbered stage labels | Each stage has its own character: WHY = brain, GUIDE = map |
